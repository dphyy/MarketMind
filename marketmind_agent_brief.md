## 1. Project Overview

MarketMind is an **autonomous merchant intelligence agent** for small-to-medium online sellers (Shopee, Lazada). It monitors competitor listings 24/7, reads social sentiment, and makes real-time pricing and ad-bid decisions — all while strictly respecting the seller's hard business guardrails.

### Core loop (runs every 15 minutes)

```
Bright Data scrapes competitor listings
        ↓
SenseNova U1 reads listing images for visual signals
        ↓
Sentiment pipeline scores social signals per product category
        ↓
Signal Aggregator normalises everything into one context object
        ↓
Guardrail Pre-check (deterministic — no AI)
        ↓
Kimi K2.6 reasons over signals → proposes mode + action + plain-English explanation
        ↓
Guardrail Post-check (deterministic — no AI)
        ↓
Action executed (simulated for hackathon) + logged
        ↓
Morning Brief generated by Kimi at 7am
```

### Four strategy modes

| Mode | Trigger | Behaviour |
|---|---|---|
| **HARVEST** | Competitor stock critically low OR positive sentiment spike | Raise price toward ceiling, increase ad bids |
| **HOLD** | Price war detected OR sentiment neutral | Hold price, moderate ad spend |
| **BUNKER** | Your own stock approaching reserve threshold | Raise price to slow sell-through, cut ad spend |
| **GHOST** | Conflicting/anomalous signals | Freeze all changes, observe only, alert seller |

## 5. Backend — Spring Boot

### `AgentCycleService.java`

This is the main orchestrator. When triggered (via HTTP endpoint or scheduler), it runs the full loop:

```java
// Pseudocode — implement fully
public AgentCycleResult runCycle(String productId) {
    // 1. Scrape competitors via BrightDataService
    List<CompetitorSnapshot> snapshots = brightDataService.scrapeCompetitors(productId);

    // 2. Extract visual signals via SenseNovaService for each snapshot with an image
    for (CompetitorSnapshot s : snapshots) {
        List<String> visualSignals = senseNovaService.extractVisualSignals(s.getImageUrl());
        s.setVisualSignals(visualSignals);
    }

    // 3. Get sentiment from SentimentService (Kimi batch classify or mock fallback)
    SentimentScore sentiment = sentimentService.getSentiment(product.getCategory());

    // 4. Build signal context
    SignalContext context = signalAggregatorService.aggregate(product, snapshots, sentiment);

    // 5. Guardrail pre-check
    guardrailEngine.preCheck(context); // throws GuardrailException if immediate violation

    // 6. Kimi reasoning → StrategyDecision
    StrategyDecision decision = kimiReasoningService.reason(context);

    // 7. Guardrail post-check on proposed action
    GuardrailResult guardrailResult = guardrailEngine.postCheck(decision, context);

    // 8. Execute or block
    if (guardrailResult.isAllowed()) {
        actionExecutorService.execute(decision);
        actionLogRepository.save(ActionLog.executed(decision));
    } else {
        guardrailBlockRepository.save(GuardrailBlock.from(decision, guardrailResult.getReason()));
        actionLogRepository.save(ActionLog.blocked(decision, guardrailResult.getReason()));
    }

    return AgentCycleResult.from(decision, guardrailResult);
}
```

### `GuardrailEngine.java`

**This is deterministic Java logic — no AI involved. Build this carefully.**

```java
public class GuardrailEngine {

    // Pre-check: validate the current state before reasoning
    public void preCheck(SignalContext ctx) {
        Product p = ctx.getProduct();
        if (p.getStock() <= p.getStockReserveMin()) {
            // Force BUNKER mode regardless of AI recommendation
            ctx.forceMode(StrategyMode.BUNKER);
        }
    }

    // Post-check: validate the proposed action from Kimi
    public GuardrailResult postCheck(StrategyDecision decision, SignalContext ctx) {
        Product p = ctx.getProduct();

        // Rule 1: Price floor
        if (decision.getProposedPrice() != null &&
            decision.getProposedPrice() < p.getPriceFloor()) {
            return GuardrailResult.blocked(
                "Proposed price $" + decision.getProposedPrice() +
                " is below margin floor of $" + p.getPriceFloor()
            );
        }

        // Rule 2: Price ceiling
        if (decision.getProposedPrice() != null &&
            decision.getProposedPrice() > p.getPriceCeiling()) {
            return GuardrailResult.blocked(
                "Proposed price exceeds ceiling of $" + p.getPriceCeiling()
            );
        }

        // Rule 3: Daily velocity cap
        double currentDailyChange = actionLogRepository
            .sumDailyPriceChangePct(p.getId(), LocalDate.now());
        double proposedChange = Math.abs(
            (decision.getProposedPrice() - p.getYourPrice()) / p.getYourPrice()
        );
        if (currentDailyChange + proposedChange > p.getMaxDailyPriceChangePct()) {
            return GuardrailResult.blocked(
                "Cumulative daily price change of " +
                String.format("%.1f%%", (currentDailyChange + proposedChange) * 100) +
                " exceeds velocity cap of " +
                String.format("%.0f%%", p.getMaxDailyPriceChangePct() * 100)
            );
        }

        // Rule 4: Stock reserve
        if (decision.getActionType() == ActionType.INCREASE_AD_SPEND &&
            p.getStock() <= p.getStockReserveMin() * 1.2) {
            return GuardrailResult.blocked(
                "Cannot increase ad spend — stock approaching reserve minimum"
            );
        }

        return GuardrailResult.allowed();
    }
}
```

```
HARVEST triggers when:
  - competitorStockLevel == OUT_OF_STOCK OR CRITICALLY_LOW
  - AND (sentimentScore24h > 0.5 OR viralityFlag == true)
  - AND stockDaysRemaining > 7

BUNKER triggers when:
  - stockDaysRemaining <= 5 OR stock <= stockReserveMin * 1.2

GHOST triggers when:
  - sentimentScore24h is null (data unavailable)
  - OR competitorPriceDeltaPct changes by more than 30% in one cycle (anomaly)

HOLD is the default when no other mode triggers
```

## 9. Database Schema

```sql
-- Products
CREATE TABLE products (
    id VARCHAR(50) PRIMARY KEY,
    name VARCHAR(255) NOT NULL,
    category VARCHAR(100),
    your_price DECIMAL(10,2),
    price_floor DECIMAL(10,2),
    price_ceiling DECIMAL(10,2),
    stock INTEGER,
    stock_reserve_min INTEGER,
    max_daily_price_change_pct DECIMAL(5,4),
    current_mode VARCHAR(20) DEFAULT 'HOLD',
    ad_bid DECIMAL(10,2),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Competitor snapshots (append-only time-series)
CREATE TABLE competitor_snapshots (
    id BIGSERIAL PRIMARY KEY,
    product_id VARCHAR(50) REFERENCES products(id),
    competitor_name VARCHAR(255),
    price DECIMAL(10,2),
    stock_indicator TEXT,
    stock_level VARCHAR(30),
    visual_signals TEXT[],
    image_url TEXT,
    data_source VARCHAR(20) DEFAULT 'LIVE', -- LIVE or MOCK
    scraped_at TIMESTAMP DEFAULT NOW()
);

-- Sentiment events
CREATE TABLE sentiment_events (
    id BIGSERIAL PRIMARY KEY,
    category VARCHAR(100),
    score_24h DECIMAL(4,3),
    trend VARCHAR(50),
    virality_flag BOOLEAN DEFAULT FALSE,
    virality_source VARCHAR(100),
    top_signal TEXT,
    recorded_at TIMESTAMP DEFAULT NOW()
);

-- Action log
CREATE TABLE action_log (
    id BIGSERIAL PRIMARY KEY,
    product_id VARCHAR(50) REFERENCES products(id),
    action_type VARCHAR(50),
    from_value TEXT,
    to_value TEXT,
    executed BOOLEAN DEFAULT FALSE,
    guardrail_blocked BOOLEAN DEFAULT FALSE,
    block_reason TEXT,
    kimi_explanation TEXT,
    daytona_job_id VARCHAR(100),
    created_at TIMESTAMP DEFAULT NOW()
);

-- Guardrail blocks (denormalised for fast UI queries)
CREATE TABLE guardrail_blocks (
    id BIGSERIAL PRIMARY KEY,
    product_id VARCHAR(50) REFERENCES products(id),
    action_type VARCHAR(50),
    proposed_value TEXT,
    block_reason TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Row-level locking: always SELECT FOR UPDATE on products row before any price/mode update
-- This prevents race conditions between concurrent agent cycles
```
